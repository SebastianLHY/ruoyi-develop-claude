---
description: 初始化项目文档结构，生成需求、状态和待办清单模板
---
 
请在当前项目根目录下执行以下操作：
 
1. 检查是否存在 `docs` 文件夹，若不存在则创建。
2. 在 `docs` 文件夹下创建以下三个 Markdown 文件，并填入对应的初始模板内容：
 
### 文件 1: `docs/需求文档.md`
请写入以下内容：
\```markdown
## 功能需求
- [ ] [编号/ID]: [事项名称]
	- 优先级: [优先级等级，如：高/中/低]
	- 预计时间: [预计耗时]
	- 状态: [当前状态，如：待开发/进行中/已完成]
	- 描述: [关于该事项的详细说明]
	- 验收标准:
	  - [ ] [具体的验收条件 1]
	  - [ ] [具体的验收条件 2]
	  - [ ] [具体的验收条件 3]
\```
 
### 文件 2: `docs/项目状态.md`
请写入以下内容：
\```markdown
# 项目状态
最后更新: [当前时间 YYYY-MM-DD HH:mm]
## 📊 当前状态
- 项目阶段: [例如：开发中 / 联调阶段]
- 整体进度: **[XX]%** [变化趋势，如：↑ (+5%)]
- 下一步计划: [当前最高优先级的进行中任务]
## ✅ 已完成
- [x] [任务名称] ([完成日期 YYYY-MM-DD])
	- 耗时: [X天/X小时]
	- 说明: [简要描述功能点]
	- Git: `[相关的 Commit Message]`
  ... (其他已完成任务)
## 🚧 进行中
- [ ] [任务名称] (开始: [开始日期])
	- 进度: [XX]%
	- 预计完成: [YYYY-MM-DD]
	- 当前工作: [具体正在做的细节]
	- 位置: `[文件路径]`
... (其他进行中任务)
## 📋 待办
- [ ] [任务名称] 🆕
	- 优先级: [高/中/低]
	- 预计工作量: [X天/X小时]
	- 模块: [模块名]
... (其他待办任务)
## 📈 里程碑
(保留此区域，除非逻辑上有变更，否则不要删除手动添加的内容)
- [x] [里程碑名称] ([完成日期])
- [ ] [里程碑名称] (预计 [YYYY-MM-DD])
\```
 
### 文件 3: `docs/待办清单.md`
请写入以下内容：
\```markdown
# 待办清单事项
## 🔥 高优先级（紧急重要）
- [ ] [任务名称]
	- 截止日期: [YYYY-MM-DD]
	- 描述: [任务具体描述]
## 📌 中优先级（重要不紧急）
- [ ] [任务名称]
	- 截止日期: [YYYY-MM-DD]
## 💡 低优先级（可延后）
- [ ] [任务名称]
	- 截止日期: [YYYY-MM-DD]
## 🔄 进行中
- [ ] [任务名称]（进度: [XX]%）
	- 开始日期: [YYYY-MM-DD]
	- 预计完成: [YYYY-MM-DD]
## ✅ 最近完成
- [x] [任务名称] (完成于 [YYYY-MM-DD])
	- 耗时: [X天/X小时]
\```
 
创建完成后，请告知“文档初始化完成”。
